## What you’re editing

The Handling tab edits entries inside `CHandlingDataMgr/HandlingData`.

Each vehicle maps to a `handlingName` entry.

## Sections

- **Identity**: `handlingName`
- **Physics**: mass, drag, percent submerged, center of mass, inertia
- **Drive**: power, top speed, gears, drive bias
- **Braking**: brake force, handbrake force, front bias, steering lock
- **Traction**: traction curves, lateral grip, spring delta, bias, camber, low-speed loss
- **Suspension**: stiffness, damping, travel limits, anti-roll, ride height, roll center
- **Damage**: collision, weapon, deformation, engine, fuel, oil
- **Seat Offset**: X/Y/Z driver seat position offsets
- **Flags & AI**: monetary value, model/handling/damage flags, AI handling profile
- **Sub Handling**: rear pop-up impulse tuning values

## Performance estimates

The editor can show rough “performance estimates” to help you iterate faster.

These are not game-accurate simulations; they’re intended as directional feedback.

## Common tuning patterns

- **More grip**: raise Max/Min traction, reduce traction loss mult
- **More drift**: lower Min traction, raise low-speed traction loss, reduce drive bias front (toward RWD)
- **Less roll**: increase anti-roll + suspension force, then dial damping to avoid bouncing
